Examen_bot

## VICEGRAM

`vicegram_bot.py` — отдельный бот-ловец для групповых **и личных** чатов
(по образцу Catcher): ловит удалённые и изменённые сообщения, стикеры,
файлы, гифки и другие медиа, следит за исчезающими медиа, восстанавливает
историю чата, считает "огоньки" активности, банит скам-сообщения и
поддерживает кастомные команды в чатах.

### Запуск

```bash
export VICEGRAM_BOT_TOKEN="токен_от_BotFather"
# необязательно: чат/канал для тихой проверки существования сообщений в группах
# (по умолчанию используется личка с VICEGRAM_ADMIN_ID)
export VICEGRAM_LOG_CHAT_ID="-100..."
python3 vicegram_bot.py
```

Добавь бота в группу с правами администратора (минимум — удаление сообщений
и чтение истории чата) и напиши в группе `/settings`.

**Техническое ограничение Bot API в группах:** Telegram никогда не присылает
боту событие "сообщение удалено". Ловля удалений реализована через
периодическую тихую проверку существования сообщения (задержка до
~30 секунд), а не мгновенно. Изменённые сообщения, медиа, анти-скам,
огоньки и команды работают штатно и мгновенно.

### Личные чаты (ловля удалений в переписках с другими людьми)

Реализовано через официальную функцию Telegram **«Автоматизация чатов»**
(Telegram Business Bots) — никакого MTProto, номеров телефона или сторонних
библиотек не требуется. Пользователь подключает бота сам, прямо в своих
настройках Telegram:

1. Настройки → «Изменить»
2. «Автоматизация чатов»
3. Вписать username бота (например `@Helperchat_bot`) → «Добавить»
4. Выбрать, каким чатам бот будет доступен («Все личные чаты, кроме…» или
   «Только выбранные»)

После подключения Telegram сам присылает боту события `business_message`,
`edited_business_message` и `deleted_business_messages` — включая
настоящее мгновенное событие удаления **с указанием чата**, в отличие от
групп, поэтому здесь не нужны ни polling, ни эвристики. Никакого
дополнительного кода на сервере настраивать не нужно — это всё уже
работает из коробки в `vicegram_bot.py`.

Похоже, что эта функция может требовать Telegram Premium на аккаунте,
который подключает бота (это часть набора функций Telegram Business) —
если пункта «Автоматизация чатов» нет в настройках, скорее всего дело в
этом.

### Пробный период и рефералы

Каждый новый пользователь получает 7 дней бесплатно с момента первого
`/start` (или с момента, когда он стал владельцем настроек группы/подключил
личные чаты). За каждого друга, который запустил бота по персональной
реферальной ссылке (кнопка «🎁 Пригласить друга» в меню) — +2 дня. Когда
пробный период заканчивается, ловля удалённых/изменённых сообщений
останавливается, а владелец получает уведомление (не чаще раза в сутки)
со ссылками на приглашение друзей и оформление подписки.

### Платная подписка

Два тарифа × четыре срока — выбираются прямо в меню бота (тариф → срок →
способ оплаты):

| | VICEGRAM MAX (всё, включая группы) | VICEGRAM PRO (всё, кроме групп) |
|---|---|---|
| 1 месяц | 59₽ | 49₽ |
| 6 месяцев | 219₽ | 199₽ |
| 1 год | 390₽ | 319₽ |
| Навсегда | 599₽ | 439₽ |

Цена в Telegram Stars — 1:1 к рублю. При годовой подписке в интерфейсе
показывается шутка про шаурму 🥙. Тарифы, сроки и цены — в константах
`SUBSCRIPTION_TIERS`, `SUBSCRIPTION_DURATIONS` и `SUBSCRIPTION_PRICES_RUB`
в начале `vicegram_bot.py`.

PRO не даёт добавлять бота в группы — если у владельца группы истёк MAX
(или он вовсе на PRO), ловля удалений/изменений в группе останавливается,
а владельцу приходит предложение проапгрейдиться. Личные чаты (через
«Автоматизацию чатов») доступны на обоих тарифах.

Три способа оплаты, все настраиваются независимо друг от друга — если
не задать переменную окружения, соответствующая кнопка оплаты просто не
показывается (кроме Stars — она встроена в Bot API и включена всегда):

- **⭐ Telegram Stars** — работает из коробки, без каких-либо ключей.
  Telegram сам обрабатывает платёж, боту приходит `successful_payment`.
- **💎 Крипта через @CryptoBot** — нужен токен приложения из
  [@CryptoBot](https://t.me/CryptoBot) → Crypto Pay → Create App:
  ```bash
  export VICEGRAM_CRYPTOBOT_TOKEN="токен_из_CryptoBot"
  ```
- **🏦 Перевод администратору** — просто текст с реквизитами, которые
  увидит плательщик; подтверждение — вручную кнопкой в личке админа:
  ```bash
  export VICEGRAM_MANUAL_PAYMENT_INFO="Перевод на карту 2200 .... по СБП, затем пришли чек сюда"
  ```

### Администрирование

ID админов заданы через переменную окружения (по умолчанию — три ID,
переданных при разработке):

```bash
export VICEGRAM_ADMIN_IDS="7928720775,1326779223,8601892147"
```

Админам в личке с ботом доступны команды:

- **`/admin`** — панель со статистикой, поиском пользователя (продление
  подписки, бан/разбан), рассылкой всем пользователям и списком последних
  платежей. Там же кнопка «ℹ️ Инструкция» с подробным описанием каждого
  пункта.
- **`/stats`** — та же статистика одной командой, без открытия панели.

Когда кто-то оплачивает переводом (🏦), чек с кнопками «Подтвердить»/
«Отклонить» приходит сразу всем админам в личку — нажать может любой,
кто увидит первым.
