Examen_bot

## VICEGRAM

`vicegram_bot.py` — отдельный бот-ловец для групповых **и личных** чатов
(по образцу Catcher): ловит удалённые и изменённые сообщения, стикеры,
файлы, гифки и другие медиа, следит за исчезающими медиа, восстанавливает
историю чата, считает "огоньки" активности, банит скам-сообщения и
поддерживает кастомные команды в чатах.

### Запуск

```bash
export VICEGRAM_BOT_TOKEN="токен_от_BotFather"
# необязательно: чат/канал для тихой проверки существования сообщений в группах
# (по умолчанию используется личка с VICEGRAM_ADMIN_ID)
export VICEGRAM_LOG_CHAT_ID="-100..."
python3 vicegram_bot.py
```

Добавь бота в группу с правами администратора (минимум — удаление сообщений
и чтение истории чата) и напиши в группе `/settings`.

**Техническое ограничение Bot API в группах:** Telegram никогда не присылает
боту событие "сообщение удалено". Ловля удалений реализована через
периодическую тихую проверку существования сообщения (задержка до
~30 секунд), а не мгновенно. Изменённые сообщения, медиа, анти-скам,
огоньки и команды работают штатно и мгновенно.

### Личные чаты (ловля удалений в переписках с другими людьми)

Обычный бот вообще не видит переписки между двумя другими людьми — это
ограничение самого Telegram, а не Bot API конкретно. Чтобы ловить удаления
и правки в **личных** чатах, VICEGRAM умеет подключать MTProto-сессию
(Telethon), залогиненную под аккаунтом самого пользователя — тогда он
видит свои диалоги как реальный участник, в реальном времени.

Пользователь подключает свой аккаунт сам, прямо в диалоге с ботом,
командой `/connect` — вводит номер телефона, потом код из Telegram (и
пароль 2FA, если он включён). Никакой отдельный скрипт для этого не нужен.

Чтобы включить эту возможность, оператору бота нужно один раз
зарегистрировать приложение на [my.telegram.org](https://my.telegram.org)
(раздел API Development Tools) и задать переменные окружения:

```bash
export VICEGRAM_API_ID="12345"
export VICEGRAM_API_HASH="0123456789abcdef0123456789abcdef"
# необязательно: свой ключ шифрования session-строк (сгенерировать:
# python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())")
# если не задать — ключ выводится из VICEGRAM_BOT_TOKEN
export VICEGRAM_ENCRYPTION_KEY="..."
```

Без этих переменных `/connect` просто скажет пользователю, что функция не
настроена — остальной бот при этом продолжает работать как обычно.

**Важно про безопасность:** после входа Telethon-session — это полный
доступ к аккаунту пользователя (как если бы кто-то держал его телефон в
руках). Она хранится в БД в зашифрованном виде (Fernet), но всё равно
является критичным секретом на уровне сервера — защищайте `vicegram.db`
как любой другой файл с паролями. Никогда не пересылайте код подтверждения
из Telegram кому-либо, кроме диалога с ботом для собственного аккаунта —
это классическая схема угона аккаунта, и бот её не отменяет, только
использует по прямому назначению.

**Протокольный нюанс:** событие удаления сообщения в личных чатах (в
отличие от групп/каналов) не содержит id чата — только id сообщения.
VICEGRAM определяет чат эвристикой по кэшу (последнее недавнее сообщение с
таким id) — работает на практике, но не даёт стопроцентной гарантии в
редких случаях совпадения id между разными диалогами.
